# 🎯 ระบบคลังข้อสอบออนไลน์: แนวทางการจัดการแต่ละโหมด

เอกสารนี้สรุปการออกแบบ logic และวิธีจัดเก็บข้อมูลในแต่ละโหมดของระบบคลังข้อสอบออนไลน์ เพื่อให้สามารถ copy มาแปะให้อ่านภายหลังได้

---

## 🔹 โหมด 1: Practice Mode (ฝึกตามหัวข้อ)

| รายละเอียด | แนวทาง |
|------------|--------|
| การทำโจทย์ | ทำเรียงตามหัวข้อ (topic) ทีละข้อ หรือหลายข้อ |
| การแสดงผล | เฉลยทันที, มีสถิติความถูกต้องรายหัวข้อ |
| การวัดผล | - บอกว่า "ผ่านหัวข้อนี้แล้ว" หากผ่านเกณฑ์ เช่น ≥ 80% จากโจทย์ทดสอบเฉพาะหัวข้อ 5 ข้อ  
| การเก็บข้อมูล | - จำนวนที่ทำแล้วต่อหัวข้อ  
- จำนวนที่ถูก  
- หัวข้อที่ "ผ่าน" แล้ว  
| การใช้ attribute | ใช้สำหรับวิเคราะห์ระดับพื้นฐานเท่านั้น (ไม่เก็บเป็น weak) |
| weakAttrScore | ❌ ไม่เก็บ  |

---

## 🔸 โหมด 2: Adaptive Mode (เรียนตามจุดอ่อน)

| รายละเอียด | แนวทาง |
|------------|--------|
| จุดประสงค์ | วิเคราะห์จุดอ่อนรายบุคคล เพื่อแนะนำโจทย์ที่เหมาะสม |
| โจทย์ที่ใช้ | ปรับตาม topic + difficulty ของผู้เรียน ไม่อิง score |
| เริ่มต้น | ใช้ "แบบทดสอบวัดความถนัด" หรือผลสอบจาก Competitive Mode เพื่อตั้งต้น weakAttrScore |
| การเก็บข้อมูล | เก็บผลตอบสนองของผู้เรียนเป็นรายโจทย์ (questionId, isCorrect)  
ใช้เพื่ออัปเดต weakAttrScore ต่อ attribute  |
| การวิเคราะห์จุดอ่อน | เก็บคะแนนต่อ attribute → นำไปใช้สุ่มโจทย์ |
| weakAttrScore | ✅ เป็นโหมดเดียวที่เก็บและอัปเดต weakAttrScore จริง |
| วิธีอัปเดต | - ตอบผิด → -1 
- ตอบถูก → +1  
- ถ้ามีหลาย attribute → หักทุกอัน  
- เริ่มต้นทุก attr ที่ 0  
- ≥ 0 ถือว่า "พ้นจุดอ่อน" |
| ระดับความยากแบบ RPG | ระบบจะปรับระดับ difficulty ของโจทย์ตามค่าคะแนน weakAttrScore:  
- +5 → diff 1:40%, 2:30%, 3:20%, 4:10%  
- +10 → diff 1:30%, 2:40%, 3:20%, 4:10%  
- +20 → diff 1:10%, 2:30%, 3:40%, 4:20%  
- ตาราง mapping ระหว่าง score และ difficulty จะเก็บไว้แยกต่างหาก  
- แนวคิด = เหมือนเลเวลเกม RPG → ยิ่งเก่ง เจอโจทย์ยากขึ้น |

---

## 🔻 โหมด 3: Competitive Mode (สอบแข่งขัน/สอบจำลอง)

| ประเภท | 3.1: ทำทีละข้อ | 3.2: สอบจำลองข้อสอบจริง |
|--------|----------------|---------------------|
| การแสดงผล | เฉลยทันที | ตรวจทีเดียวหลังทำจบ |
| การเก็บผล | - questionId  
- isCorrect 
- answerText (optional) | เช่นเดียวกัน |
| การวิเคราะห์จุดอ่อน | ✅ วิเคราะห์หลังสอบเสร็จ (เช่น "คุณควรทบทวนหัวข้อ: พื้นที่, เศษส่วน") | ✅ เช่นกัน |
| การเก็บ weakAttrScore | ❌ ไม่เก็บ ไม่อัปเดต |
| การเก็บสถิติ | - จำนวนชุดที่ทำสอบ  
- คะแนนเฉลี่ย 5 ครั้งล่าสุด |
| การใช้ attribute | ใช้เฉพาะวิเคราะห์ภายหลัง ไม่สะสมผล |

---

## 🧠 สรุปภาพรวมทั้งระบบ

| โหมด | วิเคราะห์จุดอ่อน | อัปเดต weakAttrScore | ใช้เพื่อ? |
|------|---------------------|------------------------|-----------|
| Practice | พื้นฐานรายหัวข้อ | ❌ | ฝึกเพื่อผ่านหัวข้อ |
| Adaptive | ✅ | ✅ | เรียนตามจุดอ่อนจริง |
| Competitive | ✅ วิเคราะห์เป็นครั้ง ๆ | ❌ | วัดผล, วิเคราะห์ย้อนหลัง, สร้าง weakAttr เริ่มต้น |

---

## 📦 หมายเหตุ
- weakAttrScore จะถูกใช้ในโหมด Adaptive เท่านั้น และจะไม่ถูกแตะจากผลสอบในโหมดอื่น
- การวิเคราะห์จุดอ่อนจาก Competitive Mode ใช้ attribute ที่ "ผิด" เพื่อแจ้งหัวข้อควรฝึก แต่ไม่สะสม
- การสุ่มโจทย์ใน Adaptive จะอิงทั้ง weakAttrScore และ difficulty โดยไม่สนคะแนนของโจทย์
