## 1. ภาพรวมโปรเจ็ค

ระบบฐานข้อมูลนักเรียนและการเรียนรู้ สำหรับนักเรียน **ป.4-ป.6** เน้นวินัย พัฒนาการ และแรงจูงใจ โดยมีระบบข้อสอบและ Gamification เป็นส่วนประกอบหลัก

### Stack ปัจจุบัน
- Backend: Node.js + Express
- Database: SQLite (via Prisma)
- Frontend: Vanilla JS
- Deploy: Render.com

### Philosophy หลัก
> "ทุกวันที่มาทำ คือชนะแล้ว"
> วินัยมาก่อน → ความเก่งตามมา → ถึงมีสิทธิ์แข่ง

## 2. ผู้ใช้งาน (Roles)

### 2 Role หลัก
| Role | หน้าที่ |
|---|---|
| เด็ก | ใช้งานหลัก + แสดงรายงานให้พ่อแม่ดูในแท็บเดียวกัน |
| ครู/Admin | จัดการระบบ ดูภาพรวม อนุมัติโจทย์ |

> ผู้ปกครอง **ไม่ต้อง register** — ดูรายงานผ่านแท็บในบัญชีลูกได้เลย

### การ Register
- เด็กสมัครเอง → ได้รหัส unique
- ครูใส่รหัสห้องเรียนเพื่อดูภาพรวม
- ผู้ปกครองดูรายงานผ่านแท็บในบัญชีลูก

### ข้อมูลที่เก็บของเด็ก
- ชื่อเล่น (แสดงในระบบ)
- ชื่อ-นามสกุลจริง
- ชั้นเรียน (ป.4 / ป.5 / ป.6)
- โรงเรียน
- รหัส unique สำหรับ link กับครู

## 3. การจัดการชั้นเรียน

### ปรับชั้นประจำปี (Admin)
```
ปุ่ม "+1 ทั้งหมด" → เลื่อนชั้นทุกคนพร้อมกัน
ปุ่ม [−] [ชั้น] [+] ท้ายชื่อเด็ก → fine edit รายคน
```

### กรณีพิเศษ
| กรณี | วิธีจัดการ |
|---|---|
| เด็กซ้ำชั้น | กด − คืนชั้นเดิม |
| ป.6 จบแล้ว | กด + → ระบบ Archive อัตโนมัติ |
| ย้ายโรงเรียน | อัปเดตข้อมูลโรงเรียน ชั้นไม่เปลี่ยน |

### ปีการศึกษา
- ปิดปีละครั้ง → **Archive** ทุกอย่างไว้ดูได้ตลอด
- เด็กดูประวัติย้อนหลังได้จาก profile

```
📁 ปีการศึกษา 2568 (ปัจจุบัน)
📁 ปีการศึกษา 2567 (ป.4) — archived
```

### สิ่งที่ Archive เก็บไว้
- ประวัติการทำข้อสอบทั้งหมด
- คะแนนสะสมของปีนั้น
- อันดับและรางวัลที่ได้
- ตัวละครและ item ณ สิ้นปี

### สิ่งที่ Reset ทุกปี
- คะแนนสะสม (เริ่มใหม่ที่ 0)
- Streak
- League และอันดับ
- ตั๋วแข่งขันที่ยังไม่ได้ใช้

### สิ่งที่สะสมข้ามปี (ไม่ reset)
- ตัวละคร LV
- Item หายาก

## 5. ระบบรางวัลในบ้าน (Parent Reward)

### หลักการ
- พ่อแม่และลูก **ตกลงกันเองนอกแอป**
- พ่อแม่กด **หักคะแนน** พร้อมระบุว่าแลกอะไร
- มี **log ทุก transaction** ดูย้อนหลังได้

### Log Format
```
[📅 วันที่]   [ประเภท]    [จำนวน]   [หมายเหตุ]
29 มี.ค.     ทำข้อสอบ    +10 ⭐    คณิตศาสตร์ 8/10
29 มี.ค.     แลกรางวัล   -50 ⭐    ดูการ์ตูน 1 ชม.
```

## 6. ระบบการแข่งขัน (แยกจากระบบรางวัล)

### หลักการ
- **แยกออกจากระบบรางวัลสนิท** — คะแนนสะสมไม่หาย
- ผ่านเกณฑ์คะแนน → **ได้ตั๋วอัตโนมัติ**
- แข่งตามชั้นเรียน — fair
- สุ่มโจทย์จาก DB ตามชั้น
- Admin กำหนดความถี่และรางวัลแต่ละ Season

### รูปแบบ
- League System กลุ่มละ 10-20 คน
- ใช้ชื่อเล่น ไม่เปิดเผยชื่อจริง
- แข่งแบบ Season — รายเดือนหรือรายเทอม
- หยุดทำนานเกินกำหนด → กลับมา qualify ใหม่

## 7. Gamification

### ตัวละคร + Class ตาม LV
```
LV 1-5   🌱 นักเรียนใหม่
LV 6-15  ⚔️  นักสู้
LV 16-30 🧙 จอมเวทย์
LV 31-50 🦅 อัศวิน
LV 51+   👑 ราชันย์แห่งปัญญา
```
> ตัวละคร LV สะสมข้ามปี ไม่ reset

### Item System
- ได้จากภารกิจและ milestone
- ใช้เป็น **ตั๋วเข้าด่านพิเศษ**
- ใส่ item ระหว่างทำโจทย์ → โอกาสตอบผิดได้ 1 ครั้ง
- Item หายาก เก็บข้ามปีได้

### ด่านพิเศษ + กล่องสุ่ม
Game Loop:
```
สะสมคะแนน → ได้ Item/ตั๋ว → เข้าด่านพิเศษ
→ ผ่านด่าน → เปิดกล่องสุ่ม → รางวัล
```

Reward Pool (Admin คุมได้ realtime):
| ระดับ | โอกาส (ปรับได้) | ตัวอย่างรางวัล |
|---|---|---|
| 🔴 Legendary | ~1% | Pop Mart Box |
| 🟡 Epic | ~9% | Roblox Card 500 |
| 🔵 Rare | ~30% | Roblox Card 100 |
| ⚪ Common | ~60% | ดาวโบนัส |

> Admin ปรับ % / จำกัดจำนวน / เอารางวัลออกได้ตลอด

## 8. การแสดงผลและรายงาน

### รายงาน 3 ชุด

#### 👦 เด็กเห็น (สนุก เข้าใจง่าย)
- คะแนนสะสม / ระดับตัวละคร / Streak
- ภารกิจวันนี้ทำแล้วหรือยัง
- จุดแข็ง/จุดอ่อนรายวิชา
- Item ที่มี / ตั๋วที่รอใช้
- ประวัติแลกรางวัลในบ้าน

#### 👨‍👩‍👧 ผู้ปกครองเห็น (แท็บในบัญชีลูก)
- ความสม่ำเสมอ / พัฒนาการรายเดือน
- Radar chart แต่ละวิชา
- จุดแข็ง/จุดอ่อน จาก attribute โจทย์
- ประวัติแลกรางวัลในบ้าน
- สถานะการแข่งขัน

#### 👩‍🏫 ครูเห็น
- ภาพรวมทั้งชั้น เฉลี่ยแต่ละวิชา
- เด็กที่ต้องดูแลเป็นพิเศษ
- หัวข้อที่ทั้งชั้นทำได้แย่ → ควรสอนเพิ่ม
- Export PDF/Excel ได้

| อันดับใน League | ✅ | ✅ | ❌ |
| ภาพรวมทั้งชั้น | ❌ | ❌ | ✅ |

## 9. ระบบวัดจุดแข็ง/จุดอ่อน

### หลักการ Rolling Window
```
ดูแค่ 50 ข้อล่าสุด ของแต่ละ subtopic
→ คำนวณ %
→ กำหนดสถานะ
→ บันทึกวันที่วัดผลล่าสุดของแต่ละ subtopic
```

### เกณฑ์
```
❌ จุดอ่อน        ต่ำกว่า 60%
⚡ กำลังพัฒนา    60-75%
✅ จุดแข็ง        สูงกว่า 75%
```

### โครงสร้างโจทย์ (Tag System)
- **primary tag** — บท/หัวข้อหลักที่โจทย์ต้องการ (บังคับ 1 อัน)
- **secondary tag** — เนื้อหาเสริมที่ใช้ในโจทย์ (ไม่บังคับ)

การนับคะแนน:
```
ตอบถูก → บวกคะแนนทุก tag ที่ติด
ตอบผิด → หักคะแนนแค่ primary tag เท่านั้น
```

### รายงานตาม Topic (แยกเวลาตาม subtopic)
```
📚 คณิตศาสตร์ ป.5

บทที่ 1 — เศษส่วน
  การบวกเศษส่วน   ✅ 82%   วัดเมื่อ 1 เดือนที่แล้ว
  การลบเศษส่วน    ✅ 78%   วัดเมื่อ 2 เดือนที่แล้ว
  การคูณเศษส่วน   ❌ 45%   วัดเมื่อ 3 เดือนที่แล้ว ⚠️

บทที่ 2 — เรขาคณิต
  มุมและเส้นตรง   ⚡ 65%   วัดเมื่อ 2 สัปดาห์ที่แล้ว
  พื้นที่ปริมาตร   —        ยังไม่เคยวัด

[🔄 วัดผลใหม่]
```

### Adaptive System
```
จุดอ่อน  ❌ → ออกโจทย์หัวข้อนี้บ่อย (40%)
กำลังพัฒนา ⚡ → ออกปานกลาง (30%)
จุดแข็ง  ✅ → ออกน้อย แต่ยังออกอยู่ (30%) เพื่อไม่ให้ลืม
```

## 12. AI Gen โจทย์

### Flow
```
โจทย์ต้นแบบ (จากข้อสอบแข่งขันที่มี)
→ ส่งให้ AI generate โจทย์ใหม่ N ข้อ
→ ครู Approve / แก้ไข / Reject
→ ผ่านแล้วเข้า DB
```

### สิ่งที่ AI ต้องรักษา
```
✅ โครงสร้างโจทย์เหมือนเดิม
✅ ระดับความยากใกล้เคียงกัน
✅ primary tag เดิม
✅ คำตอบถูกต้องและสมเหตุสมผล
❌ ไม่เปลี่ยนทักษะที่วัด
```

### Input โจทย์จากกระดาษ
- พิมพ์เองผ่าน Template Excel (เพราะต้นฉบับมีลายมือเฉลยปน)
- รูปภาพประกอบ → ถ่ายรูป + AI แปลงให้สะอาด → ครู verify
- เริ่มจากโจทย์ไม่มีรูปก่อน ค่อยทำรูปทีหลัง
- กราฟฟิก/รูปภาพ → ใช้ AI gen (Midjourney/DALL-E) + remove.bg

## 14. เกม "สูตรคูณแข่งกัน" (Mini Game)

### วัตถุประสงค์
ฝึกให้สมอง **random access** — เรียกข้อมูลแบบสุ่มได้เร็ว
ไม่ใช่แค่ท่องแบบ sequential (1×7, 2×7, 3×7...)

### รูปแบบเกม — แข่งรถ + Multiple Choice
```
Map เดียว ระยะ 15 ช่อง
ผู้เล่น 2-4 คน (คีย์บอร์ดเดียวหรือคนละเครื่อง)
```

### ระบบโจทย์ผสม
```
ปกติ → โจทย์ของใครของมัน (ทุกคนทำพร้อมกัน ไม่รอกัน)
สีพิเศษ → หยุดทุกคน แย่งกันกดพร้อมกัน!
```

### ระบบสีโจทย์และผล
| สี | โอกาส | ผลเมื่อตอบถูก |
|---|---|---|
| ⬜ ปกติ | 60% | เลื่อนหน้า 1 ช่อง |
| 🟦 น้ำเงิน | 20% | เลื่อนหน้า 3 ช่อง (วิ่งกระชาก!) |
| 🟥 แดง | 10% | หยุดรถคนนำอยู่ 2 โจทย์ |
| 🌀 สุ่ม | 10% | สลับตำแหน่งรถ 2 คนแบบสุ่ม 😂 |

> สีพิเศษ = โจทย์ยากขึ้น แต่ผลตอบแทนคุ้มกว่า

### Rubber Band System
```
ยิ่งนำมาก → ยิ่งเสี่ยงโดนสีแดง/สุ่ม
ยิ่งตามหลัง → ยิ่งได้โจทย์สีดี
→ พลิกล็อกได้ตลอด ตอบถูกเยอะสุดอาจไม่ชนะ
```

### ตัวเลือก 4 ข้อ (ออกแบบให้ดักเด็ก)
```
คำตอบที่ถูก   56  ← ถูกต้อง (7×8)
ตัวหลอกที่ดี  54  ← 6×9 (สับสนได้)
               63  ← 7×9 (เลขใกล้เคียง)
               48  ← 6×8 (เลขข้างเคียง)
```

### Multiplayer
- **Local** — คีย์บอร์ดเดียว แบ่งปุ่มกัน (A, F, J, L)
- **Online** — คนละเครื่อง ใช้ Room Code เหมือน Kahoot

### เชื่อมกับระบบหลัก
```
เล่นเกม → ได้คะแนน/item
ผลเกม → นับเป็น attribute "ความเร็วคำนวณ" ใน profile
```

### ขยายในอนาคต
- เพิ่มเนื้อหาอื่น: การบวก/ลบ, คำศัพท์อังกฤษ, สะกดคำไทย
- Game Engine เดิม แค่เปลี่ยน content

## 15. Roadmap

### Phase 1 — MVP (ใช้งานได้จริงทันที)
- [ ] ฐานข้อมูลนักเรียน (register, จัดการชั้น)
- [ ] Upload คะแนน + ประมวลผล + รายงาน
- [ ] Popup/ประกาศ
- [ ] ระบบข้อสอบ + Adaptive ที่มีอยู่แล้ว
- [ ] ระบบคะแนนรายวัน + โบนัสสัปดาห์

### Phase 2 — Engagement
- [ ] Gamification (ตัวละคร, Item, กล่องสุ่ม)
- [ ] ระบบรางวัลในบ้าน (Parent Reward)
- [ ] รายงาน 3 ชุด (เด็ก/ผู้ปกครอง/ครู)
- [ ] เกมสูตรคูณ Multiplayer

### Phase 3 — Competition & Content
- [ ] Mock สอบแข่งขัน
- [ ] AI Gen โจทย์ + ครู Approve
- [ ] League แข่งขัน Season
- [ ] Analytics ขั้นสูง

### อนาคต (ถ้าขยาย)
- School Management (ตารางเรียน, เช็คชื่อ)
- Learning Platform (วิดีโอ, การบ้านออนไลน์)
- Norm ระดับประเทศ
- Tournament ระดับภูมิภาค

## 16. สิ่งที่ไม่ทำ (By Design)

```
❌ Leaderboard แบบเปิดเผยชื่อและอันดับ (กดดันเด็ก)
❌ ดาวหายถ้าทำผิด (Punishment)
❌ Timer กดดันในโหมดฝึก
❌ UI ซับซ้อน (เด็กหลงทางได้)
❌ ผู้ปกครอง register แยก (ไม่จำเป็น)
❌ Face swap รูปเด็ก (ความปลอดภัย)
❌ Skin เปลี่ยนรูปตัวละคร (cost สูง)
```
